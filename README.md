# 第一章 需求分析

随着各类短视频App的飞速发展，使得百度、阿里、腾讯等互联网巨头看到了商机，分别战略布局短视频领域，其中抖音、快手等系列短视频平台迅速崛起，给各个行业带来了全新的机遇。据《2019全国短视频创意发展研究报告》显示，“截至2018年12月，我国短视频用户规模达6.48亿，用户使用率达78.2%**[1]**。自微信小程序开始进行内部测试后，就成为了移动互联网行业的热门。根据腾讯公司2018年Q1数据，微信和WeChat合并月活跃用户数已达到10.4亿，微信占据用户使用总流量的34.0%，微信已成为居民生活中非常重要的手机应用APP **[2]**，这些为微信小程序的快速发展提供了更为广阔的空间。未来5G网络一旦商用，届时短视频和微信小程序的热度将更高。

本课题开发一款与用户之间更具有亲和度的短视频微信小程序，不仅拓宽了使用可能与使用频率，也使得开发门槛与成本降低，维护变得相对轻松，这对于短视频应用发展与完善具有深刻的应用前景与意义。

# 第二章 概要设计

## （一）小程序业务流程分析

 当用户进入小程序时，默认进入主页，可以浏览各个用户已发布的视频列表。点击视频封面可以进入播放视频，同时也可以评论视频，回复评论此时如果用户为为登录状态则会跳转到登录页面，如果用户没有账号需要注册，有账号直接登录即可，登录后进入用户主页。在播放视频的同时也可以搜索视频，不管有没有搜索到都会进入与搜索页面相匹配的视频主页。在播放视频时也可以产看视频发布者，没有登录同样需要登录，登录后可以查看视频发布者的用户主页。在用户主页和视频播放页都可以上传视频，而在用户主页时用户处于已登录状态可以直接上传然后选择bgm，而在视频播放页需要检查是否登录，未登录需要登录，登陆后自动跳回，已登录则可以直接上传视频并选择bgm。在用户主页，用户上传头像后返回用户主页看到头像发生变化，可以浏览作品，收藏，关注三栏的视频列表，点击后正常播放视频。至此短视频小程序的业务流程结束。

## （四）关键技术介绍

### 1.SpringBoot框架

SpringBoot技术简化了基于Spring的应用开发，能够非常方便快捷的创建一个Spring应用。作为一个全新开源的轻量级框架，SpringBoot为所有的Spring开发提供了快速入门的体验和大型项目中常见的非功能特性。另外SpringBoot通过集成大量的框架使得依赖包的版本冲突，以及引用的不稳定性等问题得到了很好的解决。

### 2.Redis缓存技术

数据库将数据存储在磁盘或 SSD 上，而所有 Redis 数据驻留在内存中。由于无需访问磁盘，Redis 等内存数据存储避免了查找时间延迟，因此 Redis可提供亚毫秒级的响应时间，每秒处理数百万个请求。借助 Redis，可以减少用于存储、访问和使用应用程序中的数据的代码行数，从而简化代码。

### 3.FFmpeg技术

本小程序处理音视频的ffmpeg操作如下：以给1.mp4视频文件和1.mp3音频文件合成并生成视频封面为例子。首先通过ffmpeg.exe -i 1.mp4 -vcodec copy -an 11.mp4命令消除原视频声音生成1.mp4视频文件的静音版本11.mp4，接着通过ffmpeg.exe -i 11.mp4 -i 1.mp3 -t 18 -y news.mp4命令将11.mp4视频文件和1.mp3音频文件合成为一个新的视频文件new.mp4并将视频时长设置为18秒（与视频时长匹配），最后通过ffmpeg.exe -ss 00:00:01 -y -i new.mp4 -vframes 1 new.jpg命令截取new.mp4视频文件第一秒的第一帧作为视频封面。

# 第六章 项目总结

# （一）总结

小程序采用前后端分离的开发模式，前端采用的是微信小程序的原生框架wxml、wxss、json、js，使用的是微信小程序的组件和相应属性还有微信官方提供的API接口；后端利用了基于SpringBoot框架的Java语言开发相应接口，实现服务端和客户端的http通信，同时也利用Redis技术提高数据库处理速度，ffmpeg技术处理音视频的合并与视频封面截图。

 对数据库的设计，首先通过实体属性图和实体联系图汇出E-R图来实现数据库的概念结构设计，然后将E-R图转换为相应关系模式来实现数据库的逻辑结构设计，这样就完成了数据库表的设计，最后通过配置数据库的环境并对tomcat虚拟目录加以配置以实现访问静态资源。

  小程序的测试由swagger2和微信开发者工具完成，通过swagger2在线测试小程序后端接口并在前端微信开发者工具上看到相应反应以及数据库表的相应变化完成了小程序的功能测试，保证了小程序的稳定性。
